# NenioMessageSummarize

一个适用于 SillyTavern 的自动总结扩展，用来按聊天进度生成阶段性总结，并可选择写入世界书。

## 功能

- 支持手动总结当前对话。
- 支持按消息计数自动触发总结确认。
- 支持 OpenAI 兼容接口、自定义接口和 Ollama。
- 支持设置总结风格、上下文消息数、最大 Token 数。
- 支持查看历史总结、导出总结、清空总结记录。
- 支持把总结内容写入世界书条目。

## 使用方式

1. 将扩展放入 SillyTavern 的扩展目录。
2. 在 SillyTavern 中启用扩展。
3. 打开扩展面板，配置接口地址、模型和 API Key。
4. 如果使用 Ollama，本扩展允许不填写 API Key。
5. 根据需要调整自动触发频率、总结风格和上下文条数。
6. 点击“立即总结”可以手动执行一次总结。

## 主要设置说明

- `API 地址`
  - 填写兼容 OpenAI Chat Completions 的接口地址。
- `API Key`
  - 大部分云端接口需要填写；Ollama 可留空。
- `模型名称`
  - 例如 `gpt-4o-mini`、`deepseek-chat`、`qwen2.5:7b`。
- `自动触发频率`
  - 每累计 N 条新消息后弹出总结确认。
- `总结风格`
  - 可选简短、标准、详细，或自定义提示词。
- `发送给 AI 的最近消息数`
  - 控制构建总结上下文时截取的最近消息范围。
- `上下文包含之前的总结`
  - 开启后会把上一条总结作为额外上下文发送给模型。
- `写入世界书`
  - 开启后会尝试把每次总结写入世界书条目。

## 注意事项

- 本扩展依赖 SillyTavern 当前页面环境中的全局对象与事件系统。
- 世界书写入逻辑会尽量兼容常见字段，但不同版本的 SillyTavern 仍可能存在差异。
- 如果接口不是 OpenAI 兼容格式，可能需要自行调整服务端适配。

## 建议验证

1. 打开扩展面板，确认所有中文文案正常显示。
2. 使用一次“测试连接”确认接口可达。
3. 手动总结一次，检查历史记录和导出功能。
4. 若启用世界书，确认条目是否成功生成。
